# AISprint ‚Äî Consulting Website

Static marketing site for **AISprint**, a boutique AI consulting practice working with universities, research centers, and mission-driven institutions.

## Stack

No build step, no framework. Hand-written HTML, CSS, and a tiny sprinkle of vanilla JavaScript. The site is fully static and works out of the box on any web server, GitHub Pages, Netlify, Vercel, Cloudflare Pages, or S3.

- **HTML**: 7 pages (home, about, services, how-we-help, case-studies, insights, contact)
- **CSS**: one design-system stylesheet (`css/styles.css`)
- **JS**: ~40 lines of vanilla JS (`js/main.js`) for mobile nav, scroll-reveal, and the demo contact form
- **Fonts**: Inter + Source Serif 4 via Google Fonts
- **Assets**: inline SVG graphics in `assets/` ‚Äî no stock photography, no external image dependencies

## Local preview

No build is required. Just serve the folder.

```bash

python3 -m http.server 8000

# Option 2 ‚Äî Node
npx serve .
```

Then visit <http://localhost:8000>.

## Regenerating pages

The inner pages (about, services, how-we-help, case-studies, insights, contact) are generated from `generate.py`, which uses shared nav/footer partials defined in `build.py`. If you edit content in `generate.py`, regenerate:

```bash
python3 generate.py
```

`index.html` is hand-authored and is not regenerated by the script.

## Deploying to GitHub Pages

1. Push this repo to GitHub.
2. In the repo's **Settings ‚Üí Pages**, set the source to `main` branch, `/` (root).
3. Your site will be live at `https://<your-username>.github.io/<repo-name>/` within a minute.

No workflow file is needed ‚Äî GitHub Pages serves static files directly.

### Custom domain (optional)

If you own a domain, add a `CNAME` file at the repo root containing your domain (e.g. `aisprint.ai`), and configure DNS as described in [GitHub's docs](https://docs.github.com/en/pages/configuring-a-custom-domain-for-your-github-pages-site).

## Customization

### Colors & typography

All design tokens live as CSS variables at the top of `css/styles.css`:

```css
:root {
  --ink:    #051c2c;   /* primary dark */
  --accent: #00a9f4;   /* cyan accent */
  --paper:  #f6f8fa;   /* light background */
  /* ... */
}
```

Change these to rebrand the entire site.

### Adding a case study

Edit the `CASES_BODY` block in `generate.py`, add another `<article class="impact-card">` following the existing pattern, then run `python3 generate.py`.

### Adding an insight/article

Same pattern ‚Äî edit `INSIGHTS_BODY` in `generate.py`.

### Syracuse University logo

The footer currently displays a text "SU" mark with the caption *"In collaboration with Syracuse University."* If you have written permission from SU to use their official logo, drop the image file (SVG preferred) into `assets/` and replace this block in the footer of each page (or in `build.py` for the generated pages):

```html
<div class="affiliation-mark">SU</div>
```

with:

```html
<img src="assets/su-logo.svg" alt="Syracuse University" class="affiliation-logo">
```

Then add styling for `.affiliation-logo` in `css/styles.css`.

> ‚öÝÔ∏è Syracuse University's logo and name are trademarks. Only use the official logo if you have explicit written permission, and avoid implying an official endorsement you don't have.

### Favicon

Replace `assets/favicon.svg` with your own mark.

## Contact form

The form on `contact.html` is currently a front-end-only demo ‚Äî it shows a success message but doesn't actually send anything. To wire it up for real, choose one of:

- **Formspree** ‚Äî add `action="https://formspree.io/f/YOUR_ID"` to the `<form>` tag.
- **Netlify Forms** ‚Äî add `data-netlify="true"` to the `<form>` tag (and remove the `data-demo-form` attribute).
- **Your own endpoint** ‚Äî update the JS handler in `js/main.js`.

## Browser support

Tested in recent Chrome, Safari, Firefox, and Edge. Uses modern CSS (grid, custom properties, clamp) and the IntersectionObserver API with a graceful fallback for older browsers.
